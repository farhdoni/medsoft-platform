## Главный ответ

**Можно ли прямо сейчас использовать админку для управления реальными пациентами и врачами?**

**ЧАСТИЧНО — ДА, с оговорками.**

Страница «Пациенты» работает полностью: можно смотреть список, искать, добавлять, редактировать и удалять пациентов. Это готово к боевому использованию.

Страница «Врачи» работает наполовину: врачей можно добавлять и удалять, но кнопка «Редактировать» содержит ошибку — вместо сохранения изменений она создаёт нового врача-дублика.

Страница «Приёмы» — только просмотр. Создать запись к врачу через админку нельзя.

Страница «Клиники» — нельзя редактировать уже созданную клинику.

Дашборд показывает живые цифры из базы данных, но график активности пустой.

**Критично:** Страницы дашборда физически доступны без авторизации — Next.js не имеет middleware-защиты, только API проверяет токен. Если пользователь зайдёт напрямую на `/dashboard` без куки, страница откроется (и сразу покажет пустые данные из-за 401 от API), но само существование страниц не защищено.

### /patients

**Что видит пользователь:**
Таблица пациентов с колонками: ФИО, телефон, статус (цветной бейдж), группа крови, дата создания. Строка поиска (по имени, телефону). Пагинация (по 20 записей). Кнопки «Карандаш» и «Корзина» у каждой записи. Кнопка «Добавить» открывает форму с полями: ФИО, телефон, email.

**Что можно делать прямо сейчас:**

## Аутентификация и безопасность

- **Magic link:** Реализован полностью. Email с ссылкой отправляется через SMTP (Gmail) или выводится в логи сервера (режим `mock`). Токен хранится в Redis 15 минут, одноразовый.
- **TOTP 2FA:** Реализован полностью. Принудительно активируется при первом входе. Используется библиотека `speakeasy` (TOTP RFC 6238). Без TOTP войти нельзя — это не опционально.
- **JWT хранение:** Access-токен (15 минут) и refresh-токен (7 дней) хранятся в `httpOnly` куках — **правильно**, JavaScript на странице их не видит. Временный токен во время 2FA хранится в `sessionStorage` браузера — это менее безопасно, но допустимо для данного сценария.
- **Rate limiting:** Реализован через Redis. Magic-link: 5 запросов за 5 минут. Проверка TOTP: 10 попыток за 5 минут. Если Redis недоступен — лимит не применяется (fail-open — осознанное решение).
- **Защита маршрутов:** API полностью защищён — каждый маршрут требует валидный JWT с активной сессией. **Но**: Next.js (фронтенд) **не имеет** `middleware.ts` — страница `/dashboard` физически доступна без авторизации. Без куки API вернёт 401 и данные не загрузятся, но HTML-оболочка страницы откроется. Нет автоматического редиректа на `/auth/login`.
- **CORS:** Настроен на API (список доменов из переменной `CORS_ORIGINS`). Учётные данные (`credentials: true`) передаются корректно.
- **Error boundary:** Реализован в `error.tsx` (глобальный) с кнопкой «Попробовать снова».
- **Роли:** Структура ролей есть (superadmin / admin / viewer), роли проверяются для маршрута `/v1/admins` (только superadmin). Остальные маршруты — любой авторизованный администратор.
