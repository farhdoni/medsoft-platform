## Подробно по страницам

### /dashboard
**Что видит пользователь:** 6 карточек с реальными числами (пациенты, врачи, клиники, приёмы за месяц, активные SOS, выручка). Два блока с активностью за последние 7 дней: последние приёмы и SOS-вызовы. Данные обновляются каждые 30 секунд.

**API:**
- `GET /v1/dashboard/stats` — ✅ реальные данные из БД
- `GET /v1/dashboard/activity` — ✅ реальные данные, последние 7 дней

**Что можно делать:** Видеть актуальное состояние системы. Замечать SOS-вызовы.

### /patients
**Что видит пользователь:** Таблица пациентов с поиском и пагинацией. Кнопки «Добавить», «Редактировать», «Удалить» для каждого.

**API:** `GET/POST/PATCH/DELETE /v1/patients` — ✅ все работают

**Что можно делать:** Полное управление пациентами.

### /doctors
**Что видит пользователь:** Таблица врачей с поиском. Поля: ФИО, специализация, телефон, статус, рейтинг. Кнопки «Добавить», «Редактировать», «Удалить».

**API:** `GET/POST/PATCH/DELETE /v1/doctors` — ✅ все работают

**Что можно делать:** Полное управление врачами. Редактирование сохраняет именно этого врача (баг с дубликатами исправлен).

### /appointments
**Что видит пользователь:** Таблица приёмов с именами пациентов и врачей. Кнопка «Создать запись» открывает форму с выпадающими списками пациентов, врачей, клиник. Можно выбрать тип, статус, дату, цену, жалобу.

**API:** `GET/POST/PATCH/DELETE /v1/appointments` — ✅ все работают

**Что можно делать:** Создавать, редактировать, удалять записи к врачам. Менять статус приёма.

### /clinics
**Что видит пользователь:** Таблица клиник с поиском. Тип и статус с цветными значками. Кнопки «Добавить», «Редактировать», «Удалить».

**API:** `GET/POST/PATCH/DELETE /v1/clinics` — ✅ все работают

**Что можно делать:** Полное управление клиниками.

## Аутентификация и безопасность

- **Magic link:** ✅ Gmail SMTP настроен, письма приходят реально
- **TOTP 2FA:** ✅ Обязателен при первом входе, проверяется на каждую сессию
- **JWT хранение:** httpOnly cookie (`access_token` 15 мин + `refresh_token` 7 дней)
- **Защита маршрутов:** ✅ Server-side auth guard — 307 редирект на `/auth/login` без cookie
- **Rate limiting:** ✅ На magic-link (5/5мин) и verify-totp (10/5мин)
- **Аудит логи:** Таблица `audit_logs` в БД

## Что нужно доделать перед полным запуском aivita.uz

### 🟡 Важно (первый месяц)

1. **Транзакции — только чтение** (`/transactions`): Нет возможности создать транзакцию вручную. Для MVP достаточно, но нужно добавить ручной депозит и refund.
2. **SOS-вызовы — частичное управление** (`/sos-calls`): Можно назначить оператора, но нет отслеживания бригады. Нужно для продакшен-запуска экстренной службы.
3. **Поиск в /appointments**: Поисковая строка присутствует в UI, но API не передаёт параметр `search`. Нужно добавить фильтрацию.

### 🟢 Желательно (потом)

1. Экспорт данных в Excel/CSV
2. Email-уведомления при новом SOS
3. Переключение на Resend.com (лучше Gmail SMTP для продакшена)
4. Двухфакторная сессия — сейчас TOTP проверяется только при входе, не на каждый запрос
