# Виджет погоды для лендинга (aivita.uz)

Самодостаточный блок (HTML + CSS + JS), без зависимостей. Данные берёт из публичного
роута API `GET https://api.aivita.uz/v1/weather/public` — ключ OpenWeather остаётся на сервере.

## Установка (через ваш компьютер на сервер)

1. Откройте `weather-widget.html` и скопируйте весь его контент (блок `<div>` + `<style>` + `<script>`).
2. На сервере отредактируйте `/var/www/aivita-landing/index.html` и вставьте скопированное
   в нужное место внутри `<body>` (например, в шапку/герой-секцию).
3. Сохраните — отдельной сборки лендингу не требуется (статический HTML).

Пример заливки файла с локальной машины:

```bash
scp -i "C:\Users\farho\.ssh\id_ed25519" weather-widget.html root@109.123.249.224:/tmp/
# затем на сервере вручную вставить нужный фрагмент в index.html
```

## Обязательное условие на стороне API

Виджет работает в браузере, поэтому API должен разрешать домен лендинга через CORS.
Добавьте домены лендинга в переменную `CORS_ORIGIN` API и перезапустите `apps/api`:

```
CORS_ORIGIN=https://admin.aivita.uz,https://aivita.uz,https://www.aivita.uz,http://localhost:3000
```

Также на API должен быть задан `OPENWEATHER_API_KEY` (иначе роут вернёт `503`).

## Поведение

- Пробует геолокацию браузера (с запросом разрешения). При отказе/недоступности
  отправляет запрос без координат — сервер определит локацию по IP, иначе отдаст
  дефолтный город (Ташкент).
- При любой ошибке виджет тихо остаётся скрытым (`hidden`), не ломая страницу.
- Ответ кэшируется на сервере (Redis, TTL по умолчанию 10 мин); публичный роут
  ограничен 60 запросами/мин на IP.
